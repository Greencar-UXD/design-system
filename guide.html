UX·UI 종합 정리노트

개념 · 프로세스 · 리서치 · 분석 — UX/UI의 기본 개념부터 디자인 프로세스, 리서치·인터뷰·관찰, 인사이트 도출까지 한 번에 정리한 학습 노트.

Part 1

UX·UI 기본 개념

1. UX와 UI

  • UX (User Experience, 사용자 경험) — 사용자가 제품·서비스·시스템을 사용하기 전·중·후에 느끼고 생각하게 되는 총체적 경험. 화면 밖 접점(브랜드 이미지, 광고, 고객지원, 매장 분위기, 포장 등)까지 포함한다.
  • UI (User Interface, 사용자 인터페이스) — 사용자가 마주하는 시각적 접점. 사람과 시스템이 서로의 언어로 소통하도록 중간에서 다리 역할을 한다.
  • UX ≠ UI — 둘은 다른 영역. UI는 UX를 이루는 요소 중 하나이며, UX가 더 넓은 상위 개념이다. 디자인 대상(웹·모바일·자동차·음성 등)에 따라 다양한 산출물로 실체화된다.

UX는 정보구조(IA)·비주얼·인터랙션·휴먼 팩터·HCI·사운드·콘텐츠 등이 겹치는 상위 개념이다. 제품 레이어는 콘텐츠 → 서비스 → UI → 인터랙션 → 사용자 순으로 쌓인다.

2. UXD의 정의와 관점

여러 기관이 UX를 조금씩 다르게 정의하지만 공통 요소가 있다.

  • 다양한 정의 — 위키피디아(이용하며 느끼는 총체적 경험), HCI개론(상호작용하며 축적하는 지식·기억·감정), ISO 9241-210, NN/g 등. 크게 ① 적용 대상 관련 ② 인지·지각·반응 관련 정의로 나뉜다.
  • 공통 요소 — 감정적 반응 · 사용의 용이성 · 효율성 · 만족도
  • 경험을 새롭게 디자인 — 제품디자인의 대상이 '제품'이듯, UX디자인의 대상은 '사용자 경험'. 현재 경험의 이슈를 찾아 새롭게 변화시키는 것.

UX 디자이너가 가져야 할 관점 — 어떤 사람(User)이, 어떤 맥락(Context)에서, 어떤 문제·어려움(Pain point & Needs)을 겪는데, 어떤 방향으로 개선·해결·도움을 줄 수 있을까? → Solution

  • 단순 솔루션이 아니라 근본적인 문제에 대한 해결책 설계
  • 문제의 본질을 이해하고 핵심 문제를 명확히 정의
  • 개선 요소를 전략적으로 선정하고 영향 범위를 판단

3. 좋은 UX란?

  • 사용자가 느끼는 보편적 고충·갈등·어려움이 없는 것
  • 실제 이용 맥락이 참작된 것
  • 결과가 명확하고 세심하게 전달되는 것
  • 콘텐츠 양·흐름 복잡성·필요 정보에 맞춰 적절한 도구·지원을 제공하는 것
  • 희생과 선택으로 경쟁사와 다른 그 서비스만의 정체성이 분명한 것
  • 좋은 기본값(good default)을 제공하고 기능·흐름·정보량이 적당한 것

사용자를 이해하는 핵심 개념

  • 동기(motive) — 행동을 일으키는 직접 요인
  • 행동(behavior) — 경험의 구체적 발현
  • 니즈(needs) — 무언가를 필요로 하는 내적 욕구
  • 고충(pain point) — 의도의 불충분, 행동 시 어려움
  • 태도(attitude) — 대상에 대한 인지적·정서적 반응

UX 피라미드 (아래 → 위로 갈수록 높은 가치): Functional(useful) → Reliable → Usable → Convenient → Pleasurable → Meaningful

4. UI의 유형

UI는 사람과 시스템이 소통하는 방식(channel)에 따라 나뉜다.

유형설명예시
PUI (Product UI)물리적 기기·제품의 인터페이스. 물리 버튼·다이얼, 즉각적 피드백 중요냉장고 터치 패널, 자동차 대시보드
GUI (Graphical UI)그래픽 요소로 상호작용. 아이콘·버튼·메뉴, 마우스·터치데스크톱 OS, 웹 브라우저, 모바일 앱
TUI (Touchscreen UI)터치스크린 직접 조작. 스와이프·핀치 줌 제스처스마트폰·태블릿, 키오스크, 터치 ATM
VUI (Voice UI)음성 명령. 핸즈프리·자연어 처리·접근성Siri·Alexa, 차량 음성 제어, 스마트홈
NUI (Natural UI)자연스러운 신체 동작 인식. 제스처·시선, 학습 곡선 낮음Xbox Kinect, 제스처 스마트TV, AR
CLI (Command Line UI)텍스트 명령어로 제어. 고급 사용자·개발자에 적합Unix·Linux 터미널, 명령 프롬프트

분류 기준은 자료마다 다르다. 1주차 강의는 LUI(Logical)·GUI(Graphic)·AUI(Auditory)·PUI(Physical)로 묶기도 한다. 핵심은 '사람과 시스템이 어떤 채널(시각·청각·물리·음성·자연동작)로 소통하는가'.

7. UXD의 특징과 인접 분야

  • 다학제·협업이 필수 — UXD는 기술(Engineering)·경영(Business)·디자인이 만나는 지점. 정체성은 사용자 대변(User Advocacy)·리서치·사용자 중심(User Driven).
  • UXUI는 '소프트 과학(인문·사회·경영)'과 '만들기(Make)'에 걸쳐 있어 실천적인 분야로 해석된다.
  • 인접 분야 — HCI(인간-컴퓨터 상호작용), 사용성 공학(Usability Engineering), 사용자 인터페이스, 서비스 디자인, 고객 경험(CX)

디자인 대상에 따른 구분 — Product D(물리적 외형) / Interface D(입출력 장치·동작) / Interaction D(상호작용·인지) / UX D(총체적 가치·경험, 디자인 리서치에 관심) / Service D(비물리적 시스템, 공공행정에 관심)

UX vs CX vs TX

  • UX (User eXperience) — 사용자와 특정 제품의 상호작용·경험
  • CX (Customer eXperience) — 브랜드와 고객의 모든 접점의 총체. 광고·브랜드 평판·가격 공정성·고객서비스·배송 등 비(非)디지털 포함
  • TX (Total eXperience) — CX + EX(직원경험) + UX + MX(다중경험). 기업의 모든 경험을 seamless하게 통합하는 전략

9. UX 디자이너

  • 필요 역량 — ① 문제해결 ② 학습능력(트렌드 따라잡기) ③ 공감능력(협업) ④ 논리적 사고(리서치 기반 설계) ⑤ 표현력·창의성 ⑥ 커뮤니케이션. UX 디자이너는 여러 직군과 협업하는 제너럴리스트.
  • 덴마크 디자인 사다리 — ① Non-Design → ② Design as Styling → ③ Design as Process → ④ Design as Strategy. 디자인이 스타일링에서 전략적 도구로 확대.
  • 직무 지식 — 비즈니스 도메인(IoT·금융·게임·이커머스·모빌리티·헬스케어), 디자인 도메인(GUI·HCI·UXD·UID·HW·공간), 리서치(정량·통계·정성), 신기술(AI·데이터 과학·UX 리서치)

디자인 프로세스 & 방법론

10. 프로세스의 본질 — 발산과 수렴

프로세스란? 투입요소(Input)를 산출요소(Output)로 변환하는 체계. 디자이너의 행위는 문제 분석(Analysis) → 종합·해결책 도출(Synthesis) → 최종 평가(Evaluation)로 이뤄지고 피드백으로 반복(Iteration)된다.

  • The Design Squiggle (Damien Newman) — "불확실·추상적 개념을 확실·물리적 대상으로 전환". 혼란스럽게 시작한 생각이 컨셉을 거쳐 단일점으로 수렴.
  • 핵심 = 발산과 수렴의 반복 (Design Funnel). 초기 아이디어를 하나의 솔루션으로 정제.
    • 발산적 사고 — 우뇌적·직관적 (브레인스토밍, 마인드맵, 스케치)
    • 수렴적 사고 — 좌뇌적·논리적 (KJ map, AEIOU, Interaction matrix)

11. 디자인 중심 프로세스 모델

① 더블 다이아몬드 (Double Diamond) — 2005년 영국 디자인카운슬. 발산·수렴을 두 번 반복.

단계성격하는 일 / 산출물
Discover (발견)발산문제 발견. 추측보다 이해. 데스크·필드 리서치, 인터뷰
Define (정의)수렴문제 정의, 집중 부분·대상 정리. 어피니티, 퍼소나, CJM
Develop (개발)발산해결 아이디어를 찾고 고도화. 반복 검토·수정
Deliver (전달)수렴디자인 산출물로 완성·구체화

앞 다이아몬드 = 올바른 문제 찾기, 뒤 다이아몬드 = 문제를 올바르게 해결. 그 앞에 '이해하기(Understand)' 단계로 내/외부 환경을 조사·계획. 2019년 반복성 화살표가 추가되어 출시 후에도 끊임없이 피드백·개선.

② 디자인 씽킹 (Design Thinking) — 2005년 스탠퍼드 d.school.

  • d.school 모델 — 공감하기 → 문제정의 → 아이데이션 → 프로토타입 → 테스트 (5단계, 유기적 반복)
  • IDEO — 발견·해석·발상·실험·개선의 5단계('열린 다이아몬드형')
  • 변화 — 2016 NN/g가 Implement(실행) 추가, 2019 d.school이 6번째 ASSESS(평가) 추가

③ 구글 디자인 스프린트 — 2015년 구글. 이해 → 발산 → 결정 → 프로토타입 → 평가를 5일 안에(7명 팀 + 결정권자 1명). 결과보다 과정의 인사이트에 집중.

④ IIT Model / HCD

  • IIT Model — 디자인 혁신 7단계. 선형적이지 않고 피드백·개선 반복(펼치면 더블 다이아몬드와 유사).
  • HCD (인간 중심 디자인) — 관찰→맥락 이해→요구사항 정의→솔루션 생성→평가의 순환 프로세스. IDEO 버전은 Hear → Create → Deliver. 원칙: 초기부터 사용자 관점, 관찰·경청, 평가·개선·테스트 연속.

12. 개발(운영) 중심 프로세스 모델

워터폴 (Waterfall) — 절차·계획 기반 전통 방식. 한 단계가 끝나면 되돌리기 어려워 철저히 문서화.

애자일 (Agile) — '스프린트'라는 짧은 개발주기를 반복하며 개선. 민첩성·효율 극대화.

Agile Manifesto — 공정·도구보다 개인과 상호작용 / 문서보다 작동하는 SW / 계약보다 고객과의 협력 / 계획보다 변화 대응

AgileWaterfall
장점빠르고 유연, 결함을 빠르게 식별·수정, 수평적 조직순서가 정해져 따르기 쉬움, 안정적, 예산·리스크 통제 용이
단점숙련된 스크럼 마스터 필요, 잦은 변경 검토의 번거로움속도 느리고 유연성 부족, 이슈가 후반(테스트)에야 발견

린 스타트업 / 린 UX — 도요타 Lean 기법 영감(Eric Ries). 빠른 실험·피드백·개선, MVP 지향. Lean UX는 융합팀이 협업·병렬로 일하며 가설→MVP→반응 관찰→빠른 개선을 반복, UX 디자이너는 퍼실리테이터.

통합 활용 (DT + Lean + Agile) — 앞단계는 디자인 씽킹으로 탐색, 솔루션 구체화 후엔 린·애자일로 짧게 반복 개선. 프로젝트 성격에 맞게 취사선택.

13. 디자인 씽킹 & 제품 방법론 — 핵심 관점

디자인 씽킹은 디자이너의 귀납적 추론·창의 프로세스를 문제 해결 전반(전략·비즈니스·서비스·정책)에 적용하는 개념적 멘탈 모델.

  • 방법론/사이클 — 생각(Design Thinking, Double Diamond) / 연구(User Persona, A/B Test) / 제품 사이클(Agile, Lean UX, Design Sprint)
  • 창의 = (새로운 관점 + 시뮬레이션 능력 + 표현력) + (기억 + 경험 + 감성 + 지식)
  • 디자인을 잘한다는 것 = 관점 + 맥락 + 사고 패턴 + 경험 패턴
  • "방법론으로 디자인하는 게 아니라, 방법론을 디자인해야 한다"
  • Data Informed Design — 데이터로 판단하되 주관성·해석 여지를 남긴다
  • Customer Job (JTBD) — 고객이 어떤 상황인가 / 무엇을 원하는가 / 어떤 결과를 얻는가
Part 3

UX 리서치

14. 리서치의 개념과 전제

  • 리서치는 UXD 프로세스의 첫 단계(Discover)이며 발산적. 섣불리 결론 내리지 말고 일정 시점까지 계속 모은다.
  • 전제 — 문제 정의·질문이 선행. 점점 구체적 질문으로 좁힌다(퍼널): 대상자 → 진행방식 → 내용설계 → 일정계획.
  • 탐색적 리서치(무엇이 불편한가?) ↔ 평가적 리서치(우리 제품을 쓸 수 있는가?)
  • 정성(질문→대답)=초기 이해 / 정량(목적→파악)=최종 이해. 디자인 리서치는 질적 조사에 치중.

15. 리서치의 분류 — 5가지 축

분류 축갈래
① 자료의 성격양적(정량) ↔ 질적(정성)
② 자료수집 환경실험실 연구 ↔ 현장 연구
③ 연구 목적·단계탐색적 / 기술적 / 설명적(인과관계)
④ 논리 전개연역법 / 귀납법 / 가추법
⑤ 연구 주체일차조사 ↔ 이차조사
  • ① 양적 vs 질적 — 양적(정량): 숫자 데이터 통계 분석, 규모·빈도·가설 검증(설문·실험). 신뢰도·비교에 강하나 이유·맥락 통찰이 약함. 질적(정성): 비정량(텍스트·관찰), '왜·어떻게', 심층 통찰·새 가설. 일반화 어렵고 시간·전문성 요구.
  • ② 실험실 vs 현장 — 실험실: 통제 환경, 독립변수 조작, 높은 내적 타당도. 현장: 자연 환경 관찰, 높은 외적 타당도(변수 통제 어려움).
  • ③ 탐색적 / 기술적 / 설명적 — 탐색적(Exploratory): 초기 윤곽 파악, 질적·1차·개방적. 기술적(Descriptive, What·When·Who): 현상을 서술·수치화, 주로 정량. 설명적(Causal, Why): 원인 규명·예측, 주로 양적·가설 검증.
  • ④ 연역 / 귀납 / 가추 — 연역법: 이론→가설→데이터→검증(주로 정량). 귀납법: 데이터→패턴→이론(주로 질적, 근거이론 대표). 가추법(Abduction): 가장 그럴듯한 설명을 직관적으로 도출(찰스 퍼스).
  • ⑤ 일차 vs 이차 — 일차조사(Primary): 직접 raw data 수집(현장 조사). 실험·설문·인터뷰. 이차조사(Secondary): 기존 데이터 활용(=데스크 리서치). 저렴·빠르나 목적 불일치·독창성 저하 우려.

16. UX 리서치 방법론 맵 (지형도)

리서치 방법은 두 축으로 분류한다.

  • 가로축: Qualitative(질적·왜·어떻게) ↔ Quantitative(양적·얼마나·몇 개)
  • 세로축: Behavioral(행동적·무엇을 하는가) ↔ Attitudinal(태도적·무엇을 말하는가)
  • 제품 사용 방식: Natural / Scripted / De-contextualized / Hybrid

예) Usability Testing·User Interviews = 정성 / Eye-Tracking·A/B Testing·Analytics = 정량·행동 / Card Sorting·Surveys = 정량·태도. 인터뷰는 '질적 + 태도적'.

영역별 리서치 방법 (NN/g 분류)

Discovery & Innovation — Co-Creation Workshops, Contextual Inquiries, Diary Studies, Ethnography, Kano Analysis, Social Listening, User Interviews

User Behaviors — Click Paths, Eye Tracking, Heat Maps, Scroll Maps, Search Logs, Session Replay, Video Analytics

Usability & Evaluation — A/B Testing, Card Sorts, First-Click Tests, Tree Tests, Usability Testing, Wizard of Oz

UX & CX Metrics — CES, CSAT, SEQ, SUPR-Q, SUS, Time on Task, UEQ, Web Analytics

17. 조사 기법과 계획

  • 조사 계획 순서 — ① 데스크 리서치(이차조사: PEST·트렌드·경쟁사) → ② 일차조사(정성: 관찰·인터뷰 / 정량: 설문).
  • 삼각측량법(Triangulation) — 여러 방법(정성+정량)·출처·연구자·관점을 결합해 편향을 보완하고 신뢰성·타당성을 높임. 자원 소모·통합 해석이 복잡한 게 한계.

Say / Do / Know & Feel (접근성에 따른 분류)

접근성층위방법
명시적Say (말하는 것)설문, 인터뷰
관찰 가능Do (행동하는 것)맥락적 조사, 에스노그라피, 유저 다이어리, 관찰
잠재된Know / Feel생성적·참여적 방법, 사용성 테스트
  • 언어 기반 조사의 한계와 Sacrificial Prototype — 인터뷰·설문은 '원하는 것'을 묻지만 기존 경험·기존 제품에 갇히기 쉽다. IDEO는 이를 극복하려 Sacrificial Prototype(희생양 프로토타입)을 제시한다. 최종 제품 의도가 전혀 없는 저충실도 임시 프로토타입으로, 빠르게 가설을 검증하고 강·약점을 파악한 뒤 테스트 후 폐기('희생')해 반복 개선을 촉진.
  • 인터뷰 vs 설문(요약) — 인터뷰는 탐색적·소규모 질적, 설문은 명확한 문제·대규모 양적(자세한 비교는 Part 4). 질적 연구에서 대상자는 수동적 '응답자'가 아니라 능동적 참가자(파트너).

18. 데스크 리서치 (이차조사)

  • 정의 — 남이 다른 목적으로 이미 조사한 내용을 찾는 것. 내용별(사회·기술 변화·통계·전문지식·인구통계) / 출처별(문헌·학술지·web·신문·TV·DB).
  • 장점: 적은 비용으로 빠르게·객관성 확보. 한계: 목적의식 없으면 표류, 때로 일차조사보다 더 많은 시간·비용.
  • 좋은 이차조사 — ① 자신의 관점·목표 유지(핵심 흐리는 자료는 과감히 삭제) ② 참조를 반드시 표기(추적 가능, 지적자산 존중).
  • PEST → PESTPILO — PEST = Politics·Economy·Society·Technology. PESTPILO = PEST + Population·Lifestyle·Industry·Object.
Part 4

인터뷰 & 설문

19. 인터뷰의 이해

  • 보편적 인터뷰(뉴스 취재 등)와 달리 UX 리서치의 인터뷰는 보통 심층 인터뷰(In-Depth Interview) — 피상적 문답이 아닌 경험의 깊은 내용 파악.
  • 속성 — 질적(Qualitative) + 태도적(Attitudinal). 대화로 맥락·깊이를 추구하고, 행동 관찰보다 생각·감정·의견·경험에 초점.
  • 유의점 — ① 말과 실제 행동은 다를 수 있다 ② 무의식 행동은 기억 못함 ③ 기억력 한계로 정확한 답 기대 어려움. → 사진·영상 등 실마리로 기억·몰입 강화. 품질은 진행자·분석자의 역량에 의존.

20. 인터뷰 유형

기준유형
질문 구조구조화(질문 고정) / 반구조화(융통성 있게 가감) / 비구조화(질문 미정)
답변 형식폐쇄형(선택지 = survey형) / 개방형(자유 진술)
장소인터뷰어 공간 / 제3의 공간 / 인터뷰이 공간(맥락적 질문법 가능)

표현 수단은 글(서면 인터뷰)·대화·표정·몸짓·행동을 모두 동원한다. 특히 맥락적 질문법(Contextual Inquiry)은 사용자 환경에서 Task를 주어 행동을 관찰하고 Think aloud·사후 인터뷰를 병행한다.

참여 인원·대상별 종류

  • 전문가 인터뷰 / 헤비유저 — 정확·압축 정보를 빠르게. 섭외 어렵고 비용↑.
  • 델파이 기법(Delphi) — 인터뷰는 아니나 전문가 의견을 모아 합의점을 찾음. RAND 개발, 익명 설문·서면을 반복해 결론으로 수렴.
  • 극단적 사용자(Extreme User) 인터뷰 — 극단적 사용자에게서 시야 확장·혁신 기회. 부정/평균/긍정 1/3씩 권장.
  • FGI / FGD — 5~6인 그룹을 진행자(Moderator·Facilitator) 주도로 90분 전후. 'big mouth' 중재·사전 서면질문. 장점: 결론 도달·심도. 단점: 실제 현장 조사 불가, 생생한 맥락 누락.
  • 비대면 인터뷰 — 시간·장소 자유(해외 가능)·디지털화 용이. 단점: 인터넷·녹화 실패 시 치명적.

21. 인터뷰 진행하기

인터뷰는 준비 → 진행 → 정리 3단계.

  • 준비 — 계획(누구를 만나 무엇을 알아볼지·방법·스크립트), 리크루팅·스케줄, 사례 준비, +파일럿 테스트. 가장 큰 관건은 '누구를 만나느냐'. 보통 60~90분.

진행 단계의 질문 흐름

단계내용
① 도입 (Introduction)목적·개요 전달, 동의서·촬영/녹음 동의
② 분위기 형성 (Warm-up)가벼운 이야기로 라포(Rapport) 형성. 생략 가능
③ 스토리적 질문과거 경험을 '과정'으로 묻는 열린 질문. 감정 실린 지점 체크(Pain·Need·Satisfaction)
④ 전반적 질문 (General)주제 전반. 구조화 질문(사전 서면질의서)
⑤ 집중 질문 (Deep focus)형성된 관심 초점에 깊이 있게. '왜?'(5 Whys), 비구조화. 가장 핵심 단계
⑥ 확인 (Retrospective)내용 확인·추가 논의(필요시)
⑦ 정리 (Wrap-up)마감 인사, 사례, 추가 약속
  • 보조도구 — 행동 분석(녹화), 발화 분석(녹음). 중단이 잦으니 진행 전 반드시 테스트.
  • 인터뷰어 태도 — "인터뷰는 인터뷰어가 누구냐에 좌우된다". 경청·라포·호응, 표면 발화 이면의 근본 이유를 생각, 한 번에 하나씩.
  • 나쁜 인터뷰 4가지 — ① 무례한 인터뷰(위압) ② 대화 끊김(폐쇄형 단답) ③ 답변에 영향(유도질문) ④ 부담 주는 인터뷰(직접 해결책 요구·어려운 전문용어).

5 Whys — 도요타(1930년대). '왜?'를 반복(꼬리질문)해 근원을 찾음. 주의: 꼭 5번일 필요 없음, 사고 숙성 여유, 궁극적 연구질문을 잊지 말 것.

22. 설문조사 (Survey)

  • 언제 — 명확해진·의식적(명제화된) 문제, "몇 %가 동의하는가"를 알 때.
  • 특징 — 장점: 객관적 데이터. 단점: 생생함·유연함 결여, 탐색 단계 부적합. → 실시 전에 연구모형·가설 수립 필요.
  • 형태 — 전화 / 이메일 / 문서(오프라인) / form(온라인, 구글폼)

인터뷰 vs 설문

인터뷰설문조사
대상탐색적 문제("왜?")명확해진 문제(명제화)
특징소규모 질적(십여 명 이하, 통계 X)대규모 양적(최소 30명~수천 명, 통계처리)
관건진행자 자질·리크루팅문항·표본설계

23. 인터뷰·설문 정리(분석) 방법

처리과정 없이 결론을 적으면 '블랙박스·자의적 결론'. 발화에 라벨(코드, code)을 붙여 체계적으로 정리한다.

  • 어피니티 다이어그램 — 발화를 전사 후 관련성 높은 것끼리 그룹핑해 추상적 결론을 끌어내는 Bottom-up(= Open Card Sorting). 포스트잇에 핵심 적기 → 유사끼리 묶어 주제 형성 → 우선순위·인사이트.
  • 토픽별 분석 — FGI·인터뷰처럼 주제가 명확할 때. 토픽에 따라 발화를 셀에 기입하는 Top-down. 탐색적 과정엔 부적합.
  • 코딩스킴 — 발화를 의미 단위(코드)로 식별하는 가장 체계적인 방법. 연역적 코딩(기존 이론 기준 → 신뢰도↑·새 패턴 발견 어려움), 귀납적 코딩(데이터 기반 맞춤 → 시간↑·연구자마다 상이). 중복 코딩 가능.
  • 근거이론(Grounded Theory) — 데이터에서 이론을 도출. 개방코딩 → 축코딩 → 선택코딩 → 이론적 코딩으로 '이론적 포화' 지향.
  • 프로토콜 분석 — UT에서 발화·행동을 사전 코딩스킴으로 코딩해 인지 흐름 파악.
  • 통계처리 — 코딩 기반 빈도·교차분석 → 정성 발화를 정량 자료처럼 다룸.
Part 5

관찰 조사

24. 관찰의 이해

  • 관찰은 가장 원초적인 조사이자 혁신의 잠재력. 그러나 효과적으로 하거나 가르치기 어렵다.
  • 정의 — 대상을 지켜보고 기록·수집(= 현장 조사). 자연·물질은 관측(측정 도구·기준), 사회·사람은 관찰(통찰).
  • 분류 — 통제/비통제 관찰, 직접/간접 관찰.
  • 장점: 대상이 왜곡되지 않고, 의사소통 불가 대상도 연구 가능. 단점: 비효율적, 일반화 어려움, 연구자 편차 큼.
  • 실사용자 행동 관찰로 얻는 해결안(디자인실에선 못 얻음). 단, 관찰한다고 해결안이 항상 떠오르진 않는다. (예: OXO 계량컵 / 맥도날드 밀크쉐이크 — '아침 출근길 단품 대용' 용도 발견)

에스노그라피(Ethnography) ≒ 관찰연구

  • 디자인에서 거의 동의어. 1999년 인텔 연구자들이 'Design Ethnography' 발표. 90년대 말 웹·휴대전화 확산으로 문화적 이슈 부각.
  • 정의 — 문화인류학적 방법, 연구자가 대상 집단과 동화되어 장기 참여·관찰. "ethnos(사람)+grapho(기록)"='문화 기록'. (말리노프스키 『서태평양의 항해자』)
  • 얀 칩체이스(Jan Chipchase) — '디자인계의 인디아나 존스'. Immersion 전략: 현지인 집에 살며 생생한 organic 데이터를 얻으라고 강조.

25. 관찰의 유형

두 축 — 은폐(대상자로 봄) ↔ 노출(참가자로 봄), 비개입 ↔ 개입.

유형개입/노출설명
Fly on the wall (몰래 관찰)비개입·은폐'벽에 붙은 파리'처럼 눈에 띄지 않게 자연스러운 행동 관찰
Shadowing (동행 관찰)비개입·노출그림자처럼 따라다니며 관찰. 동의 필요, private 공간 추적 불가
Contextual Inquiry (맥락적 질문법)개입·노출활동 장소에서 행동에 대해 질문하며 관찰. 가장 본격적(인터뷰 병행)
Undercover Agent (잠입 조사)개입·은폐신분을 숨긴 채 직접 인터랙션. 디자인 리서치엔 잘 안 씀
Usability Test개입참가·예측이 가능한 중·후반 단계
Immersive Research (몰입조사)관찰자=대상관찰자가 직접 대상을 경험
User Diary (유저 다이어리)자기보고참가자가 스스로 경험을 기록·보고

26. 다양한 관찰 기법과 사례

  • Thoughtless Act — '무념의 행동'에 내재된 니즈 (예: 짐 든 채 지하철 표를 입에 물기).
  • Behavioral Archeology (행동의 고고학) — 의자 배치·낙서·잔디 마모 같은 물리적 흔적으로 행동을 추론.
  • Town Watching — Hip Place에서 장기간(4주+) 관찰·촬영. 라이프스타일·트렌드·상권 분석.
  • 카메라 관찰 — 사용자에게 카메라 지급(Self/Disposable Camera, User Picture Interview).
  • 맥락적 질문법(현장방문조사) — 사용자 환경에서 인터뷰+관찰. 계획 → 사용자 선정 → 파트너로 존중 → 현장 관찰·대화 → 구체적 대화 → 해석 검토. Think aloud·사후 인터뷰. 팀은 Facilitator + Interviewer/Recorder 2명.
  • User Diary — 참가자 자기보고형. 관찰 불가 장소·시간 조사 가능. (예: 거실 IoT 연구 — 292개 활동을 AEIOU로 재구성)
  • 몰입형 연구(Immersive Research) — 관찰자가 직접 경험. 감정이입·섭외 불필요하나 객관성 낮음. 사례: Kristian Simsarian(IDEO 응급실 입원) / Patricia Moore(3년간 노인 분장 → 유니버설·인클루시브 디자인, OXO Good Grips).
  • Behavioral Map / Tracing — 이동 동선·행동을 평면도에 표기해 공간 활용도·이동 형태 파악. 레이아웃 문제까지 찾기.
  • (Cultural) Probe — Bill Gaver 등(1997~99). 참여자에게 패키지(소책자·일회용 카메라·지도·엽서)를 주고 스스로 결과물 제작·보고. 5~10명.
  • UT 관찰 (with sacrificial prototype) — 적극 개입 관찰. 초기 단계에도 가능, Lo-fi 프로토타입·Think aloud가 효과적.

관찰의 태도 (유의점)

  • 편견 없이, 새로운 눈으로 많이 보라. 무의식적 고정관념은 극복하기 어렵다.
  • 현장 자료를 가능한 많이 기록(녹음·녹화·메모·손스케치). 다만 기록보다 '왜?' 사고가 더 중요.
  • 의미 해석·숙고를 가능한 빨리 하라. 정성 자료는 시간이 지나면 상하는 식재와 같다.
  • 제품·사용자·문화를 너무 잘 알면 고정관념에 빠지기 쉽다. 기본 룰·절차는 있으나 규정된 것은 아니다.

분석 & 인사이트 도출

28. 인사이트란 — 자료를 '정보'로

  • 더블 다이아몬드의 정의(Define) 단계 목적은 개별 자료를 의미 있는 정보로 엮어 '무엇을 디자인할 것인가'에 대한 답을 얻는 것.
  • 분석 = 정보화. 자료가 정보·지식·지혜로 격상되는 흐름이 DIKW다.
단계의미행위
자료 (Data)객관적 사실 ("컵에 물이 반 있다")연구·조사·수집
정보 (Information)표현·조직화된 자료표현, 조직화
지식 (Knowledge)해석이 들어간 것 ("반이나 / 반밖에")통합, 사고, 해석
지혜 (Wisdom)적용 가능한 통찰사고, 회상, 적용

핵심은 흩어진 자료를 연결·조직화하고, 패턴을 찾는 것이다.

29. 분석 방법 개괄

  • Bottom-up(아이디어를 모아 → 숨은 패턴·구조 발견) ↔ Top-down(전체 맥락을 명확히). 둘을 오가며 반복.
  • 분석은 정성 ↔ 정량 스펙트럼에 놓인다. 데스크·관찰·인터뷰·FGI·다이어리·프로브(정성) ↔ 설문·실험(정량).
  • 포지셔닝맵·근거이론·코딩스킴은 정성이 본질이나 수량화가 가능해 정성+정량 혼합 성격.

30. 분석 프레임워크

① AEIOU framework (Doblin) — 경험 기반 관찰 결과를 해석하는 틀. Activity·Environment·Interaction·Object·User. ⚠️ 프레임워크만으로는 통찰이 나오지 않는다.

② POEMS framework — 경험의 여러 측면을 포괄 이해. People·Objects·Environment·Messages·Services.

AEIOUPOEMS
초점사용자 행동·환경 관찰, 경험 흐름경험 구성요소·서비스 관계 정리
쓰임사용성 평가·서비스 개선제품/서비스 개발·경험 설계
  • ③ 어피니티 다이어그램 (KJ법) — 가와키타 지로 개발, 대표적 정성 분석. 1단계 노트 만들기(생생한 문장, 발화자 표기) → 2단계 그룹핑(유사끼리) → 3단계 의미 추출(노랑=노트, 파랑=그룹명, 분홍=상위 개념).
  • ④ 포지셔닝맵 / 이미지맵 — 경쟁 제품·브랜드 위치를 2축 평면에 시각화. 이미지맵은 컬러(웜↔쿨, 하드↔소프트) 기준. 정량+정성 모두 포함.
  • ⑤ Entity Relation Matrix — 일리노이공대 찰스 오웬. 상향식·요소환원주의. 요소 나열 → 관계 3단계(강함>있음>없음) → 다이어그램 배치. 확장하면 Network Diagram.
  • ⑥ 다이어그램 — 글보다 이해·기억·전달에 우수하고 한눈에 파악·개념화하는 장점.
  • ⑦ 맥락적 질문법의 산출물 5가지 (카렌 홀츠블렛) — Flow(관계·흐름)·Sequence(작업 방식·시간순)·Artifact(도구 구조)·Physical(작업 환경)·Cultural(문화적 요인) model.

부록

부록 — 사용성 평가 · 논문

부록 A. 사용성 평가 (Usability Test)

평가 유형 — 탐색 평가(근본 기능·멘탈 모델 파악) / 측정 평가(태스크 수행·수행시간·오류 수 정량) / 검증 평가(출하 전 기준 확립·증명) / 비교 평가(대안 비교).

프로세스 (5단계) — ① 평가 계획(목적·대상·방법·기준·참가자·태스크) → ② 사전 평가(Pilot, 1~2명 예행) → ③ 평가 진행(오리엔테이션→소개→태스크→설문→면담) → ④ 결과 분석(패턴화·어피니티) → ⑤ 평가 보고서(재현 가능하게 기술).

세부 — 평가 기준 · 참가자 · 태스크 · 진행

평가 기준 (유효성·효율성·만족도 각 최소 1개) — 유효성(성공 비율·정확도) / 효율성(완수 시간) / 만족도(불만 빈도 등)

참가자 — 실제+잠재 사용자. 프로필(성별·나이·학력·직업·사용 경험·환경). 그룹: 초급/중급/고급

태스크 — 대표 태스크(기능적·오류적·시간적). 작게 하되 실제처럼, 목표 명확히. 총 1시간~1시간 30분 권장.

진행 인원 — 진행자 / 시간 / 기록 / 비디오 기록 / 관찰자

유의 — 진행자는 견해 표현 자제, "참가자"로 호칭. 평가 대상은 사용자가 아니라 제품·소프트웨어. 도중 문제가 생겨도 도움을 주지 않는 것이 원칙.

  • 일정 — 4학기: 논문 발표(3차 준비) · 3학기: 1·2차 발표, 게재지 등재 시 통과 간주
  • 참고 — RISS, 디자인리서치학회, 한국연구재단 등재지
